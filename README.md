# نظام إدارة مركز صيانة السيارات

نظام ويب متكامل مبني بـ **Flask (Python) + SQLAlchemy** بتصميم **SAP Fiori** ودعم كامل للغة العربية (RTL).

## المتطلبات
- Python 3.10+
- (اختياري) SQL Server إذا أردت استبدال قاعدة SQLite الافتراضية

## خطوات التشغيل (أول مرة)

```bash
# 1) تثبيت الحزم
pip install -r requirements.txt

# 2) تهيئة قاعدة البيانات (تُنشئ الجداول + مستخدم افتراضي + بيانات أساسية)
python seed.py

# 3) تشغيل النظام
python run.py
```

سيعمل النظام على: **http://localhost:5080**

### بيانات الدخول الافتراضية
- اسم المستخدم: `admin`
- كلمة المرور: `admin123`

> يقوم `seed.py` أيضًا بتفعيل **ترخيص تجريبي لمدة سنة** تلقائيًا على هذا الجهاز حتى تتمكن من تجربة النظام مباشرة دون الحاجة لإدخال مفتاح تفعيل يدويًا.

## الاتصال بـ SQL Server بدلاً من SQLite
عدّل متغير البيئة `DATABASE_URL` قبل التشغيل:

```bash
export DATABASE_URL="mssql+pyodbc://USER:PASSWORD@SERVER/DBNAME?driver=ODBC+Driver+17+for+SQL+Server"
```

يتطلب ذلك تثبيت `pyodbc` و`ODBC Driver 17/18 for SQL Server` على الجهاز، ثم تشغيل `python seed.py` مجددًا لإنشاء الجداول على قاعدة SQL Server.

## هيكل المشروع
```
garage_system/
├── app/
│   ├── models/          # جميع جداول قاعدة البيانات (SQLAlchemy)
│   ├── blueprints/       # الشاشات (كل شاشة = Blueprint مستقل: routes.py)
│   ├── services/         # منطق الأعمال (المخزون، الترخيص، التصدير، الترقيم)
│   ├── templates/        # قوالب Jinja2 بتصميم SAP Fiori
│   └── static/           # CSS / الخطوط / الصور المرفوعة
├── config.py              # إعدادات قاعدة البيانات والتطبيق
├── run.py                 # نقطة التشغيل (منفذ 5080)
└── seed.py                # تهيئة أولية لقاعدة البيانات
```

## الميزات المنفذة
- تسجيل دخول/خروج + تذكرني + صلاحيات حسب الدور لكل شاشة (عرض/إضافة/تعديل/حذف/طباعة/تصدير)
- لوحة تحكم بكل المؤشرات المطلوبة (سيارات، مخزون، مبيعات، مشتريات، تنبيهات نقص)
- إدارة كاملة للسيارات وأوامر الصيانة + تسجيل الأعطال + تركيب قطع الغيار مع خصم تلقائي من المخزون ومنع الصرف عند نقص الكمية
- إدارة قطع الغيار (كود تلقائي، فئات، استيراد Excel)
- إدارة المخازن + التحويل بين المخازن + الجرد الفعلي + تسوية المخزون + تقرير الوارد والمنصرف
- الموردين وفواتير الشراء مع تحديث المخزون تلقائيًا
- تقارير احترافية (11+ تقرير) مع معاينة + تصدير Excel + تصدير PDF بدعم عربي كامل + طباعة
- إدارة المستخدمين والأدوار مع صلاحيات تفصيلية لكل شاشة
- إعدادات الشركة، تغيير كلمة المرور، نسخ احتياطي/استعادة لقاعدة SQLite
- نظام تفعيل واشتراك (مفتاح مشفّر HMAC مرتبط باسم الشركة وبصمة الجهاز، مدد 6 أشهر/سنة، إيقاف تلقائي عند الانتهاء)

## ملاحظات مهمة
- هذا مشروع أساسي متكامل وقابل للتشغيل فعليًا (تم اختباره Route بـ Route)، لكن نظرًا لضخامة النطاق المطلوب (نظام ERP كامل)، بعض الشاشات الثانوية (مثل استيراد فئات القطع من Excel أو شاشات مطابقة كل تقرير من الـ 20+ تقرير بتصميم مخصص) مبنية بأسلوب عام (Generic) قابل للتوسعة بسهولة بدلاً من شاشة مخصصة منفصلة لكل حالة.
- في بيئة إنتاج حقيقية: غيّر `SECRET_KEY` في `config.py` و`SECRET_KEY` في `app/services/license_service.py`، واستخدم خادم WSGI حقيقي (gunicorn/waitress) بدلاً من خادم التطوير.
